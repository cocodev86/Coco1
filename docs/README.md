## Document hierarchy

2. Operating standards

4. Client operations
5. Platform specifications
6. Strategic roadmap
7. Reusable implementation templates

## Governance

Every controlled document includes an identifier, version, owner and status. Material changes should be reviewed through pull requests. Architecture decisions belong in ADRs; product requirements belong in PRDs; implementation designs belong in TDDs.
